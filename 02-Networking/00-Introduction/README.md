# 📖 Chapter Overview

Computer networking is the foundation of modern computing. Whether you're browsing the web, sending an email, connecting to cloud services, or performing a penetration test, you're interacting with a network.

Before learning IP addressing, routing, protocols, or network security, it's essential to understand the fundamental concepts that make communication between devices possible.

This chapter introduces those concepts in a logical, beginner-friendly sequence. Each lesson builds upon the previous one, creating a strong foundation for every networking topic that follows throughout this roadmap.

# 🛡 Why Networking Matters in Cybersecurity

Almost every cybersecurity activity depends on networking.

Whether you're capturing packets with **Wireshark**, scanning a network using **Nmap**, investigating malware communications, analyzing firewall logs, or defending enterprise infrastructure, you're working with network traffic.

Understanding networking enables you to:

- Discover how attackers move across networks.
- Understand how defenders monitor network activity.
- Troubleshoot communication problems.
- Analyze malicious traffic.
- Secure systems and services.
- Build a strong foundation for penetration testing, digital forensics, cloud security, and SOC operations.

> **💡 Remember**
>
> Cybersecurity is not just about hacking computers—it's about understanding how computers communicate.

## 📘 Lesson 4 — [Client-Server Architecture](Client%20Server%20Architecture.md)

> **The communication model that powers the modern internet.**

Most websites, cloud platforms, online games, and business applications follow the **Client-Server** model. In this architecture, one device requests a service (the client), while another device processes the request and provides the response (the server).

Understanding this model helps explain how the majority of modern network services operate.

### You'll Learn

- What a client is
- What a server is
- How client-server communication works
- Real-world examples of client-server architecture
- Advantages and limitations

### Why This Lesson Matters

From web servers and databases to authentication systems and cloud platforms, the client-server model is everywhere. It is one of the most important networking concepts you'll encounter throughout your cybersecurity journey.

📍 **Next:** Compare this centralized architecture with a decentralized communication model.

---

## 📘 Lesson 5 — [Peer-to-Peer Architecture](Peer%20to%20Peer%20Architecture.md)

> **A decentralized way for devices to communicate.**

Unlike the Client-Server model, a **Peer-to-Peer (P2P)** network allows devices to communicate directly with one another without relying on a dedicated server.

Although less common in enterprise environments, peer-to-peer communication remains an important networking concept and is widely used in file sharing, blockchain technologies, and distributed systems.

### You'll Learn

- How Peer-to-Peer networks work
- Differences between P2P and Client-Server
- Advantages and disadvantages
- Common real-world use cases

### Why This Lesson Matters

Understanding both communication models helps you recognize how different networks are designed and prepares you for future topics such as network services, cloud computing, and distributed applications.

📍 **Next:** Learn how organizations classify networks based on ownership and accessibility.

---

## 📘 Lesson 6 — [Internet vs Intranet vs Extranet](Internet%20vs%20Intranet%20vs%20Extranet.md)

> **Understanding public and private network environments.**

This lesson brings together everything you've learned by exploring how networks are categorized based on **accessibility**, **ownership**, and **purpose**.

You'll learn the differences between the public Internet, private organizational Intranets, and controlled-access Extranets.

### You'll Learn

- What the Internet is
- What an Intranet is

- Key differences between them
- Real-world organizational examples

### Why This Lesson Matters

Cybersecurity professionals constantly work with public and private networks. Understanding these environments is essential before learning protocols, services, and network security concepts.

# 💡 Study Tips

Networking is best learned by **understanding concepts**, not by memorizing definitions.

As you progress through this roadmap:

- Draw simple network diagrams.
- Visualize how devices communicate.
- Practice identifying network types around you.
- Think about which topology a real network might use.
- Compare Client-Server and Peer-to-Peer examples you use every day.
- Ask yourself **why** a technology is designed a certain way instead of simply remembering its name.

The stronger your networking fundamentals become, the easier every cybersecurity topic will be.
